# NetPractice

The `NetPractice` project is part of the 42 Network curriculum. It is an interactive project designed to help students understand the fundamentals of **computer networking** through hands-on exercises.

The goal is to configure small network scenarios by correctly setting **IP addresses**, **subnet masks**, and **routing tables**.

---

## Description

NetPractice (42cursus) - The purpose of this project is to develop a strong understanding of networking fundamentals by solving practical exercises involving IP addressing, subnetting, routing, and network configuration in a simulated environment.

---

## Features

- **IP Addressing**
  - Understand IPv4 structure
  - Configure valid IP addresses

- **Subnetting**
  - Work with subnet masks
  - Divide networks efficiently

- **Routing**
  - Configure routing between networks
  - Understand gateways and paths

- **Network Troubleshooting**
  - Identify misconfigurations
  - Fix connectivity issues

- **Interactive Learning**
  - Solve progressively difficult levels
  - Immediate feedback on configurations

---

## Getting Started

### Access the Project

NetPractice is done directly in your browser via the 42 platform.

---

## Usage

1. Open the NetPractice interface.
2. Each level presents a **network topology**.
3. Your task is to:
   - Assign correct IP addresses
   - Set subnet masks
   - Configure routing if needed
4. Click **Check again** to validate your solution.

---

## Concepts Covered

- IPv4 Addressing
- Subnet Masks
- CIDR Notation
- Routing Basics
- Default Gateway
- Network Segmentation

---

## Example
IP: 192.168.1.10
Mask: 255.255.255.0
Gateway: 192.168.1.1


---

## Rules

- All devices must be correctly configured
- Networks must be reachable
- No invalid IP addresses
- Subnet masks must match network requirements

---

## Technical Details

- **Type**: Networking project
- **Environment**: Web-based simulator
- **Focus**: Logic and configuration (no coding required)

---

## Tips

- Always check if devices are in the same subnet
- Use binary thinking for subnetting
- Verify gateway correctness
- Break down problems step by step
